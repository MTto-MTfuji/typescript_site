# スマホ版テストガイド

スマホ版の表示をテストする方法を説明します。

## 📱 テスト方法

### 方法1: ブラウザの開発者ツールでテスト（最も簡単）

**Chrome/Edgeの場合:**

1. **開発サーバーを起動**
   ```bash
   npm run dev
   ```

2. **ブラウザで開く**
   - 通常は `http://localhost:5173` で開きます

3. **開発者ツールを開く**
   - `F12` キーを押す
   - または `Ctrl + Shift + I`（Windows）/ `Cmd + Option + I`（Mac）

4. **モバイル表示モードに切り替え**
   - 開発者ツールの左上にある「デバイスツールバー」アイコンをクリック
   - または `Ctrl + Shift + M`（Windows）/ `Cmd + Shift + M`（Mac）

5. **デバイスを選択**
   - 上部のドロップダウンから「iPhone 12 Pro」「Galaxy S20」などを選択
   - または、カスタムサイズを設定

6. **テスト項目**
   - ✅ ハンバーガーメニューが表示されるか
   - ✅ メニューが開閉できるか
   - ✅ レイアウトが崩れていないか
   - ✅ ボタンがタップしやすいサイズか
   - ✅ 広告が正しく表示されるか（設定済みの場合）
   - ✅ テキストが読みやすいか

**Firefoxの場合:**

1. `F12` で開発者ツールを開く
2. 右上の「レスポンシブデザインモード」アイコンをクリック
3. デバイスを選択

---

### 方法2: 実際のスマホでテスト（ローカルネットワーク経由）

**同じWi-Fiに接続している必要があります**

1. **開発サーバーを起動**
   ```bash
   npm run dev
   ```

2. **PCのIPアドレスを確認**
   
   **Windowsの場合:**
   ```bash
   ipconfig
   ```
   - 「IPv4 アドレス」を確認（例: `192.168.1.100`）

   **Macの場合:**
   ```bash
   ifconfig | grep "inet "
   ```
   - `192.168.x.x` のアドレスを確認

3. **スマホでアクセス**
   - スマホのブラウザで `http://192.168.1.100:5173` にアクセス
   - （IPアドレスは実際の値に置き換えてください）

4. **注意事項**
   - PCとスマホが同じWi-Fiに接続されている必要があります
   - ファイアウォールの設定によっては接続できない場合があります

---

### 方法3: Vercelにデプロイしてからテスト（最も確実）

1. **Vercelにデプロイ**
   - GitHubにプッシュ
   - Vercelでデプロイ（環境変数は既に設定済み）

2. **デプロイ完了後**
   - Vercelが提供するURL（例: `https://your-project.vercel.app`）を確認

3. **スマホでアクセス**
   - スマホのブラウザでURLを開く
   - 実際のユーザーと同じ環境でテストできます

## 🐛 よくある問題と解決方法

### 問題1: スマホでアクセスできない（方法2の場合）

**原因**: ファイアウォールがブロックしている

**解決方法**:
- Windows: コントロールパネル → ファイアウォール → ポート5173を許可
- Mac: システム環境設定 → セキュリティ → ファイアウォール設定

### 問題2: メニューが開かない

**原因**: JavaScriptエラーが発生している可能性

**解決方法**:
- ブラウザのコンソール（F12）でエラーを確認
- `npm run build` でビルドエラーがないか確認

### 問題3: レイアウトが崩れている

**原因**: CSSが正しく読み込まれていない

**解決方法**:
- ブラウザのキャッシュをクリア
- `npm run dev` を再起動

## 📝 テスト後の確認事項

テストが完了したら、以下を確認してください：

1. **問題がない場合**
   - ✅ そのままデプロイに進めます

2. **問題がある場合**
   - 問題を修正
   - 再度テスト
   - 問題が解決したらデプロイ
